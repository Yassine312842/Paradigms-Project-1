# TCP Chat Application – Backend

## Overview

This project implements the **backend (logical layer)** of a multi-client chat application using **Java TCP sockets**.
The server allows multiple clients to connect simultaneously, exchange messages, and execute basic chat commands.

The networking logic is intentionally separated from any graphical interface so that a **UI (e.g., JavaFX)** can later interact with the backend without modifying the networking code.

This repository therefore focuses on:

* TCP client-server communication
* Multi-client handling using threads
* Command processing
* Message broadcasting
* Clean backend architecture

---

## Project Structure

The repository contains two Maven projects.

### TCPServer

Responsible for managing the chat server and client connections.

Key classes:

* **TCPServer**
  Application entry point. Loads configuration and starts the server.

* **ServerModel**
  Core server logic.

    * Accepts client connections
    * Maintains the list of connected clients
    * Broadcasts messages

* **ClientHandler**
  Handles communication with a single client using a dedicated thread.

Configuration file:

```
TCPServer/src/main/resources/server.properties
```

Example configuration:

```
server.ip=127.0.0.1
server.port=3000
```

---

### TCPClient

Console-based client used to test the backend.

Key classes:

* **TCPClient**
  Entry point for the client application.

* **ClientModel**
  Handles networking logic such as:

    * connecting to the server
    * sending messages
    * receiving messages asynchronously

The client implementation serves as a **testing interface** for the backend and demonstrates how a graphical interface could interact with the server.

---

## Features

The backend supports the following functionality:

* Multiple clients connected simultaneously
* Broadcast messaging between clients
* Username identification
* System notifications when users join or leave
* Read-only client mode
* Basic command handling
* External server configuration
* Thread-safe client management

---

## Commands

Clients can use the following commands.

### allUsers

Returns the list of currently connected users.

Example:

```
allUsers
```

Server response:

```
[12:45] (SYSTEM): Active users: Alice, Bob
```

---

### bye / end

Disconnects the client from the server.

Example:

```
bye
```

---

## Read-Only Mode

If a client connects without providing a username, the server assigns a temporary identity:

```
READ_ONLY_<port>
```

Read-only users:

Allowed:

* receive messages
* request the user list

Not allowed:

* send chat messages

---

## Message Format

Messages follow a consistent format generated by the server.

Normal messages:

```
[HH:mm] username: message
```

Example:

```
[14:21] Alice: Hello everyone
```

System messages:

```
[HH:mm] (SYSTEM): message
```

Example:

```
[14:22] (SYSTEM): Bob joined the chat
```

## How to Run the Application

### Start the Server

Navigate to the folder containing the server JAR and run:

```
java -jar TCPServer.jar
```

Example output:

```
Server started on 127.0.0.1:3000
Waiting for clients...
```

---

### Start a Client

Open another terminal and run:

```
java -jar TCPClient.jar localhost 3000
```

The client will ask for a username.

Example:

```
Enter username (leave empty for READ-ONLY mode):
```

Multiple clients can connect simultaneously.
